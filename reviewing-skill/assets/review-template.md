# Findings:

List the issues found during the review.

For each finding, include:
- Severity: High, Medium, or Low
- A natural explanation of the issue and its impact
- The file and line reference embedded in the sentence
- Prefer including a short quoted phrase from the reviewed text when useful

Use one finding per bullet.
Do not use the literal label `Why it matters:`.
If there are no issues, write `No findings.`

# Bottom line:

State what must be fixed before the skill should be relied on.
This can be blank if nothing is a must-fix issue.

# Suggestions:

List suggested improvements.

Include design pattern recommendations here when they are useful but not strictly must-fix.
If any of the five design patterns applies, say so explicitly.

# Answers:

Answer any additional user questions here.
If there were no other questions, say `None.`

End with a direct question to the user.
Name the main suggested changes explicitly instead of saying only `the suggestions`.
